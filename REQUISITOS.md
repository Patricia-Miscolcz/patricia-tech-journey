# Requisitos do Projeto LoginCloud

## Requisitos Funcionais

- RF01: O sistema deve permitir que o usuário informe e-mail e senha.
- RF02: O sistema deve validar os dados informados no login.
- RF03: O sistema deve permitir o cadastro de novos usuários.
- RF04: O sistema deve consultar os dados em um banco de dados em nuvem.
- RF05: O sistema deve exibir mensagens de erro para credenciais inválidas.

## Requisitos Não Funcionais

- RNF01: A interface deve ser responsiva.
- RNF02: O sistema deve ser simples e intuitivo.
- RNF03: O código deve ser versionado no GitHub.
- RNF04: O projeto deve utilizar GitHub Projects para organização das tarefas.
- RNF05: A aplicação deve ser compatível com navegadores modernos.

## Regras de Negócio

- O usuário só poderá acessar o sistema se possuir cadastro válido.
- O e-mail deverá ser único para cada usuário.
- A senha deverá ser obrigatória para autenticação.

## Critérios de Aceitação

- O arquivo REQUISITOS.md deve estar criado na raiz do projeto.
- O documento deve conter requisitos funcionais e não funcionais.
- O documento deve estar versionado no GitHub.
